# StaritsinPartners

Настольное приложение на WPF + .NET + Entity Framework Core + PostgreSQL для учёта партнёров и продаж.

## Возможности

- ведение списка партнёров с карточками;
- добавление и редактирование данных партнёра;
- просмотр истории реализации продукции по партнёру;
- управление продажами;
- автоматический расчёт индивидуальной скидки партнёра;
- сохранение исторической цены и скидки на момент продажи;
- автоматическое создание базы данных при первом запуске;
- модульные тесты на MSTest.

## Стек технологий

- .NET 8
- WPF
- Entity Framework Core 6
- PostgreSQL
- MSTest

## Структура проекта

- `StaritsinPartners.WPF` — пользовательский интерфейс
- `StaritsinPartners.Domain` — сущности, контекст БД, репозиторий, сервисы, бизнес-логика
- `StaritsinPartners.Tests` — модульные тесты

## Настройка базы данных

Перед запуском приложения необходимо создать роль и базу данных.

**Выполнить от пользователя `postgres`:**

```sql
CREATE ROLE app WITH LOGIN PASSWORD '123456789';
CREATE DATABASE staritsin OWNER app;
```

После этого при первом запуске приложение **автоматически создаст** схему `app`, все таблицы и заполнит их начальными данными.

Либо создать схему и таблицы вручную с помощью скрипта:

```
psql -U app -d staritsin -f StaritsinDB.sql
```

## Параметры подключения

Приложение использует следующие параметры подключения к PostgreSQL:

- Host: `localhost`
- Port: `5432`
- Database: `staritsin`
- Username: `app`
- Password: `123456789`

Строка подключения:

```
Host=localhost;Port=5432;Database=staritsin;Username=app;Password=123456789
```

## Запуск приложения

Открыть решение в Visual Studio 2022 и запустить проект:

```

```

Через .NET CLI:

```
dotnet test
```

Либо запустить тесты из Visual Studio через **Test Explorer** (`Ctrl+R, A`).

## Основные бизнес-правила

- скидка рассчитывается по суммарному объёму продаж партнёра:
  - до 10 000 — 0%,
  - 10 000–49 999 — 5%,
  - 50 000–299 999 — 10%,
  - от 300 000 — 15%;
- при продаже сохраняются:
  - базовая цена товара,
  - скидка партнёра,
  - цена продажи на момент оформления;
- WPF-проект не содержит прямых зависимостей от Entity Framework Core — вся работа с БД инкапсулирована в `StaritsinPartners.Domain`.
